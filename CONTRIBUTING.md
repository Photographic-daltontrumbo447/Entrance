# Contributing

Thank you for wanting to help.

During `V0 HEADLESS ALPHA`, this repository is still stabilizing its runtime,
licensing, and commercial boundaries. Because of that, unsolicited external code
contributions are not being broadly accepted yet.

## What is welcome right now

- bug reports
- reproducible issue notes
- design feedback
- release feedback from self-hosted alpha use
- licensing and packaging feedback

## What is intentionally limited right now

- drive-by code pull requests
- relicensing-sensitive contributions into core runtime code
- trademarked forks presented as official surfaces

## If you want to contribute code

Open an issue first and describe:

1. the problem
2. the proposed change boundary
3. why it belongs in core instead of a plugin or adapter

If the change is a fit, maintainers may invite a patch under a separate written
contribution process. A formal CLA may be introduced before general code
contributions open up.

## Inbound licensing expectation

Unless explicitly agreed otherwise in writing, any contribution accepted into
this repository must be mergeable under the repository's then-current licensing
scheme and release policy.

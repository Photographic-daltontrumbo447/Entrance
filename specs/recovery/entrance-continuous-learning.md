# Entrance Continuous Learning

> This document is the Entrance-local reinterpretation of continuous learning.
> It preserves the parts we consider fundamental and absorbs them into our own memory architecture.

## 1. Role in the System

Continuous learning belongs to the memory-evolution layer.

Its job is to turn repeated observations, mistakes, preferences, and architectural clarifications into durable system knowledge.

## 2. Preserved Essence

The parts worth preserving are:

- instinct as the smallest behavioral learning unit
- confidence-aware learning rather than binary certainty
- scope awareness
  project-local signals should not automatically become global truth
- promotion pipeline
  not every raw signal deserves immediate elevation into a stable rule

## 3. Entrance Adaptation

Entrance extends the learning model beyond a single instinct table.

Recovered and ongoing memory now naturally separates into:

- `memory_fragments`
  raw or partially structured signals
- `instincts`
  behavioral patterns and action guidance
- `decisions`
  accepted structural judgments
- `visions`
  long-horizon direction
- `documents`
  canonical narrative or explanatory text
- `coffee_chats`
  reflective synthesis
- `todos`
  unresolved action items
- `memory_links`
  provenance and semantic graph structure

## 4. Multi-Source Learning

Entrance does not restrict learning to one observation channel.

Valid truth sources may include:

- Human conversation
- code review and implementation outcomes
- Linear history
- recovery fragments
- external process logs
- AntiGravity chat history
- SQL commands or operational traces that prove how a memory was originally produced

This is a major difference from narrower hook-centered learning systems.

## 5. Promotion Rule

Inside Entrance, a useful default progression is:

1. raw signal enters `memory_fragments`
2. repeated or stable behavioral guidance becomes an `instinct`
3. accepted structural judgment becomes a `decision`
4. long-horizon direction becomes a `vision`
5. broader explanation becomes a `document`
6. unresolved work becomes a `todo`

## 6. What We Reject

We do not want:

- every stable idea forced into `instincts`
- hook-only learning pipelines
- flat note-taking without provenance
- project and global scope mixing without explicit modeling

## 7. Practical Rule

Continuous learning in Entrance should mean:

- capture first
- classify honestly
- promote conservatively
- preserve provenance
- let the memory graph evolve without collapsing all knowledge into one table

## 8. Boundary

If an idea conflicts with the current Entrance framework, the framework wins.

External source material can contribute useful primitives, but it does not outrank the local architecture, product identity, or memory model.
